# Tool Grouping Configuration
#
# Format: path_prefix|module_name
#
# Maps a DCT API path prefix to the generated tool module file it belongs to.
# Used by toolsgenerator/driver.py:_get_module_for_path to decide which
# <module_name>_tool.py file each generated unified tool is written into.
#
# Matching rule: longest prefix wins (e.g. "/management/engines" beats
# "/management"). Paths with no matching prefix fall back to "misc_endpoints".
#
# NOTE: The semantics of this file differ from the pre-bbe27c0 version, which
# mapped tool_name -> path_patterns. That concern is now handled by the
# "# TOOL N: tool_name" headers in config/toolsets/*.txt. This file only
# controls the output filename grouping for generated tools.
